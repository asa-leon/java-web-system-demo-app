# 🏛️ 「机上院」を爆アゲする機能アイデア集
## 1. 🗳️ 採決システム（本会議・委員会採決）
単なる「いいね」ではなく、法案に期限（例：提出から3日間）を設け、期限到来時に「本会議」が開かれるイメージです。

- **賛成・反対・棄権の三択投票**
   - ユーザーは「賛成 / 反反対 / 棄権」を選択（意見の理由も任意で添えられる）。
- **可決・否決の判定**
  - 投票期間終了時、賛成が一定割合（例：50%超）を超えると「可決（成立）」、下回ると「廃案」のバッジが付く。
- **可決された法案の「施行」**
  - 可決された優秀な法案は「机上院・現行法律集（六法）」というアーカイブ画面に殿堂入りとして登録される。

## 2. 💬 質疑通告 ＆ 答弁システム（コメントの拡張）
コメント欄を普通の雑談ではなく、「国会審議」の形式にします。

- **「質疑通告」ボタン**
  - コメント投稿時に「一般コメント」か「質疑通告」かを選択。
  - 質疑通告にすると、提出者（議員）に「◯◯議員より質疑が通告されました」と通知が飛ぶ。
- **「政府答弁（提案者答弁）」ステータス**
  - 提案者がその質疑に返信（答弁）するまで、該当質問に「答弁待ち」の警告バッジが付く。
  - 答弁が完了すると「答弁完了」となり、質問と回答が1つの「審議議事録」として綺麗にカード表示される。

## 3. 📜 修正案（Amendment）の提出
法案の本文に対して「ここをこう変えた方がもっと良くなるのでは？」という改善案を他のユーザーが出せる機能です。
- **「修正案を提出する」**
  - 他のユーザーが、元の法案のテキストを一部変更した「修正案」を紐付けて投稿できる。
- **提案者の受け入れ（受け入れ可否）**
  - 原案作成者（法案提出者）が「修正案を採用」とすると、法案本文が書き換わるか、バージョン（v1.1）が上がる。

## 4. 🪪 ユーザーの「党派・議員ステータス」ゲーム化
投稿や投票を重ねることで、ユーザーの「議員としての権威」が育つ仕組みです。
- **政党システム**
  - ユーザー同士で「〇〇党（例：猫の幸せを考える党、エンジニア労働環境改善党）」を結成または所属できる。
- **当選回数・役職バッジ**
  - 法案が可決された数や、質疑・答弁の回数に応じて「当選1回」「委員会委員長」「国政モニター」などの称号がプロフィールに付与される。
